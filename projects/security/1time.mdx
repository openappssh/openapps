1time is a zero-knowledge secret sharing tool that creates encrypted one-time links for passwords, API keys, and sensitive text. All encryption happens in the browser using AES-256-GCM with HKDF key derivation — the server never sees plaintext, and the decryption key exists only in the URL fragment.

### Privacy by Design

- **Client-side encryption**: AES-256-GCM encryption and decryption run entirely in the browser. The server stores only ciphertext and has no way to decrypt it.

- **URL fragment isolation**: The decryption key is placed in the URL fragment (#), which browsers never send to the server. Even if the server is compromised, secrets remain encrypted.

- **No accounts, no tracking**: No signup required, no cookies set, no analytics collected. Completely anonymous usage.

### CLI and Automation

- **Pipe-friendly CLI**: `printf 'postgres://user:pass@host/db' | 1time send` creates an encrypted link from your terminal. Reads from stdin, `1TIME_SECRET` env var, or positional argument.

- **Self-hosted instances**: Point the CLI at your own deployment with `--host` flag. HTTPS enforced except for localhost.

### Built-in Generators

- Strong random passwords, memorable passphrases, WiFi-safe passwords, and API key generators — all running client-side with no server calls.
